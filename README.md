# wav_analyzer

WAVファイルの基本情報とラウドネス指標(ピーク値・RMS)を解析し、壊れている可能性のあるファイルを自動検出するコマンドラインツールです。ディレクトリを指定すると、配下のWAVファイルを再帰的に検出し、並列に解析します。

## 概要

指定したディレクトリ以下のWAVファイルをすべて検出し、各ファイルについて以下を解析します。

- サンプルレート
- チャンネル数
- ビット深度
- サンプル数
- ピーク値(絶対値の最大サンプル値)
- RMS(実効値、音量の平均的な大きさの指標)
- 警告(無音の可能性、クリップの可能性)

出力形式はテキストとJSONの2種類に対応しています。

## 使い方

### ビルド

```bash
git clone https://github.com/ユーザー名/wav_analyzer.git
cd wav_analyzer
cargo build --release
```

### 実行

```bash
# テキスト形式(デフォルト)。指定ディレクトリ以下を再帰的にスキャン
cargo run -- path/to/directory

# JSON形式
cargo run -- path/to/directory --format json
```

### 出力例

**テキスト形式**

```
見つかったWAVファイル数: 1
--- path/to/directory/test.wav ---
サンプルレート: 44100
チャンネル数: 1
サンプル深度: 16
サンプル数: 91334
ピーク値: 12158
RMS: 1226.6710641225209
```

警告が該当する場合は、末尾に `⚠ 無音の可能性があります` のような行が追加されます。

**JSON形式**

```json
{
  "sample_rate": 44100,
  "channels": 1,
  "bits_per_sample": 16,
  "samples_count": 91334,
  "peek": 12158,
  "rms": 1226.6710641225209,
  "warnings": []
}
```

## 機能一覧

- ディレクトリ再帰スキャンによるWAVファイル自動検出
- 複数ファイルの並列解析(マルチコアを活用した高速処理)
- 基本情報(サンプルレート・チャンネル数・ビット深度・サンプル数)の解析
- ラウドネス指標(ピーク値・RMS)の算出
- 無音検出(RMSが一定値未満)
- クリップ検出(ピーク値がビット深度最大値の95%を超過)
- text / json 出力形式の切り替え
- ファイルが開けない・破損している場合もパニックせず処理を継続

## 使用技術

- [Rust](https://www.rust-lang.org/)
- [hound](https://crates.io/crates/hound) — WAVファイルの読み込み
- [clap](https://crates.io/crates/clap) — コマンドライン引数のパース
- [walkdir](https://crates.io/crates/walkdir) — ディレクトリの再帰スキャン
- [rayon](https://crates.io/crates/rayon) — 複数ファイルの並列解析

## 実装のポイント

- `Result`と`?`演算子によるエラーハンドリングを一貫して採用し、ファイルが存在しない・破損している場合もパニックせず、わかりやすいエラーメッセージを表示するようにしています。1件のファイルが不正でも、他のファイルの処理には影響しません。
- 解析結果を構造体(`AnalysisResult`)にまとめることで、「計算処理」と「表示処理(テキスト/JSON)」の責務を分離しています。
- `rayon`の`par_iter()`を使い、通常のイテレータをほぼそのままの記述で並列化しています。Rustの所有権・借用の仕組みにより、データ競合のない安全な並列処理をコンパイル時に保証しています。
- `clap`の`derive`機能を使い、コマンドライン引数の定義を宣言的に記述しています。
